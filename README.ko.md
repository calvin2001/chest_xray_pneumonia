# 흉부 X-ray 폐렴 분류

*Read this in [English](./README.md).*

흉부 X-ray 이미지를 **정상(NORMAL)** 과 **폐렴(PNEUMONIA)** 으로 분류하는 딥러닝 프로젝트입니다. PyTorch로 의료 영상 분류를 처음부터 끝까지 다루며, 단순히 점수를 높이는 것이 아니라 **의료 현장의 관점으로 모델을 평가하는 것** — 즉 *폐렴을 놓치는 실수*를 가장 중요하게 다루는 것 — 에 초점을 맞췄습니다.

---

## 문제 정의

흉부 X-ray 한 장을 보고 폐렴 소견이 있는지 판단하는 이진 분류 문제입니다. 여기에는 의료 특유의 비대칭성이 있습니다. 아픈 환자에게 정상이라고 말하는 오류(거짓 음성, FN)가 헛경보(거짓 양성, FP)보다 훨씬 위험합니다. 이 비대칭성이 프로젝트의 모든 평가 판단을 이끕니다.

## 데이터

- **출처:** Kaggle [Chest X-Ray Images (Pneumonia)](https://www.kaggle.com/datasets/paultimothymooney/chest-xray-pneumonia) (약 5,856장)
- **기본 분할:** `train` (5,216장) · `val` (16장) · `test` (624장)
- **클래스 불균형:** 학습 데이터에서 폐렴이 정상보다 약 **3배** 많습니다(3,875 vs. 1,341). 그래서 정확도(accuracy)는 오해를 부르고, 대신 폐렴 클래스의 재현율(recall)을 추적합니다.
- **검증셋 주의:** 기본 `val` 폴더는 **16장**뿐이라 신뢰하기 어렵습니다. 그래서 `train`에서 계층적(stratified)으로 제대로 된 검증셋을 다시 떼어냈습니다(아래 참고).

## 접근 방법

각 단계가 다음 단계의 필요성을 이끌도록 순차적으로 구성했습니다.

1. **데이터 탐색** — 이미지가 어떻게 `[C, H, W]` 텐서가 되는지, 클래스 불균형 확인, 정상 vs. 폐렴 폐를 눈으로 비교.
2. **도메인을 존중한 증강** — 좌우 뒤집기, 작은 각도 회전(±10°), 약한 밝기 조정. 상하 뒤집기와 큰 회전은 **의도적으로 배제**했습니다. 뒤집히거나 크게 회전된 흉부 X-ray는 현실에 존재하지 않기 때문입니다. 증강은 **train에만** 적용하고, val/test에는 고정 변환만 써서 평가가 정직하고 재현 가능하도록 유지했습니다.
3. **베이스라인: 처음부터 만든 작은 CNN** — 데이터가 적어 성능이 낮을 수밖에 없는 의도적으로 약한 기준선입니다. 다음 단계의 전이학습 효과를 눈에 보이게 만드는 대조군 역할을 합니다.
4. **ResNet18(ImageNet) 전이학습** — 사전학습된 특징 추출기를 재사용하고 분류층만 다시 학습합니다. 두 방식을 비교했습니다. **특징 추출(freeze)**: 앞부분을 얼리고 `fc`만 학습 / **미세조정(fine-tune)**: 전체를 풀되 작은 학습률로 조정.
5. **과적합 대응 & 의료 평가** — 드롭아웃, early stopping, best 모델 저장, 계층적 검증셋 재구성, loss 곡선 진단, 혼동행렬의 의료적 해석, 그리고 **놓친 폐렴 사례를 직접 눈으로 보는** 오류 분석.

## 왜 recall이 핵심 지표인가

라벨은 `정상 = 0`, `폐렴 = 1` 입니다.

- **거짓 음성(FN, 놓친 폐렴)** — 아픈 환자를 치료 없이 집으로 돌려보냄. 의료적으로 가장 비싼 실수.
- **거짓 양성(FP, 헛경보)** — 정상인을 폐렴으로 표시. 추가 검사와 불안을 유발하지만, *안전한* 쪽의 실수.

거짓 음성을 최소화한다는 것은 곧 **재현율(recall)을 최대화**한다는 뜻입니다. 그래서 recall을 최우선 지표로 두고, 균형을 위해 정밀도(precision)와 F1도 함께 보고합니다.

## 결과

별도로 떼어둔 test 세트(624장: 정상 234 + 폐렴 390), 각 3 epoch, 동일한 로더 기준:

| 모델 | Recall | Precision | F1 | 놓친 폐렴 (FN) | 헛경보 (FP) |
|---|---|---|---|---|---|
| 작은 CNN (처음부터)          | 0.99 | 0.69 | 0.82 | 3 | 172 |
| ResNet18 — 특징 추출 (freeze) | 0.98 | 0.85 | **0.91** | 8 | 65 |
| ResNet18 — 미세조정 (fine-tune) | **1.00** | 0.75 | 0.86 | 1 | 128 |

계층적 검증셋 재구성 + 드롭아웃 + early stopping으로 다듬은 뒤(노트북 5장 참고), 처음부터 만든 CNN의 precision은 약 0.72(F1 약 0.83)로 올랐고, 미세조정 모델은 recall 약 1.00 / precision 약 0.76 / F1 약 0.86로 안정화됐습니다.

*수치는 Colab(Tesla T4) 실행 결과이며, 증강과 드롭아웃이 무작위라 재실행 시 조금씩 달라집니다.*

## 결과가 말해주는 것

- **전이학습이 주력이다.** 약 5천 장뿐인 데이터에서도 ImageNet 특징을 빌려오면 처음부터 학습하는 것보다 precision과 F1이 앞서고, 노력도 훨씬 적게 든다.
- **모든 모델이 매우 높은 recall(~0.98–1.00)에 도달한다.** 미세조정 모델은 폐렴을 사실상 놓치지 않는다(390장 중 1장). precision이 낮은 것(헛경보가 많은 것)은 의료 맥락에서 받아들일 만한 교환이다.
- **freeze vs. fine-tune은 형식이 아니라 실제 선택이다.** 얼린 특징 추출기는 전체 균형이 가장 좋고(F1 ≈ 0.91), 미세조정은 recall을 극대화한다. 무엇을 배포할지는 임상 목표가 "아무것도 놓치지 말 것"인지 "균형 잡힌 선별"인지에 달려 있다.
- **숫자보다 오류 분석.** 지표만 읽는 대신 놓친 폐렴 X-ray를 직접 들여다보는 것이, 점수를 "모델이 어디서 왜 틀리는가"에 대한 이해로 바꿔준다.

## 실행 방법

GPU 런타임의 Google Colab을 기준으로 설계했습니다.

1. Colab에서 `chest_xray_pneumonia.ipynb`를 열고 **런타임 → 런타임 유형 변경 → GPU**로 설정합니다.
2. Kaggle API 토큰을 발급하고(**Kaggle → Settings → API → Create New Token**), 다운로드된 `kaggle.json`을 Colab 파일 탐색기에 업로드합니다.
3. 셀을 위에서부터 순서대로 실행합니다. 초기 셀들이 Kaggle 클라이언트 설치, 데이터 다운로드·압축 해제, 폴더 구조 확인까지 처리한 뒤 학습이 시작됩니다.
